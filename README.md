# CCP-TEPP-SummerSchool-Documentation

## Session 1 - In-code documentation (with Doxygen) (1.5hrs)

**Goals**:

- Appreciate the value of *good* in code documentation
- Understand how to write Doxygen style documentation
- Hands on experience
- Knowledge of kind of tools that are available to help

Plan:

1. Intro into comments and docs
2. Setting up doxygen
3. Writing doxygen comments
4. Getting fancy
5. GitHub pages

## Session 2 - Out of code documentation (with Sphinx) (1hr)

**Goals**:

- Appreciate the need for documentation from outside of the code
- Gain familiarity with Sphinx

Plan:

1. Reminder of session 1
2. Motivate need for documentation outside of code
3. Setting up Sphinx
4. Writing a getting started page
5. Getting fancy
6. Highlight Readthedocs

## Session 3 - Types of docs (with Diataxis Framework) (1.5hrs)

**Goals**:

- Understand the breadth of documentation types
- Be able to identify missing documentation in a project
- Have ideas for how to fill these gaps

Plan:

1. Reminder of session 2
2. Look at open source packages and identify
   some kinds of documentation in the wild
3. Introduce Diataxis framework
4. Identify gaps in docs in the wild
